# Claude-sandbox

## The problem

Claude CLI has shell, filesystem, and network access on your machine. A compromised or tricked agent can read secrets, use SSH keys, abuse Docker, and exfiltrate data вҖ” not just edit your repo.

## Solutions

### Colima VM

Claude runs directly inside a dedicated **Colima Linux VM**. Isolation is enforced by the VM hypervisor вҖ” Claude shares no kernel with the host, the host Docker daemon is completely unreachable, and only `/workspace` is mounted from the host. No Docker required on the host.
